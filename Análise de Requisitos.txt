## Análise de Requisitos

### 1. Requisitos Funcionais (O que o sistema deve FAZER)

Estes são os recursos diretos que o usuário ou o administrador (você) irá interagir.

* **RF01 - Cadastro de Modelos (Templates):** O sistema deve permitir o upload de uma imagem de fundo (JPG/PNG) para servir de base para o certificado.
* **RF02 - Mapeamento de Campos:** O administrador deve conseguir definir as coordenadas $(x, y)$ onde o nome do aluno, curso e data serão impressos sobre o template.
* **RF03 - Geração em Lote:** O sistema deve ser capaz de processar uma lista de alunos (via Banco de Dados ou upload de CSV/Excel) e gerar todos os PDFs de uma vez.
* **RF04 - Envio Automático por E-mail:** Após a geração, o sistema deve disparar o PDF como anexo para o e-mail cadastrado do aluno.
* **RF05 - Validação via QR Code:** Cada certificado deve conter um QR Code único que, ao ser lido, direcione para uma página de consulta no seu domínio (ex: `mazzcursos.com.br/validar/123`).
* **RF06 - Histórico de Emissões:** O sistema deve manter um log de quem emitiu, quando e qual certificado foi gerado.

---

### 2. Requisitos Não Funcionais (Como o sistema deve SER)

Estes garantem a qualidade, segurança e performance da aplicação.

* **RNF01 - Performance:** A geração de um PDF individual não deve ultrapassar 2 segundos. Processamentos em lote (ex: 100 alunos) devem ser feitos de forma assíncrona (usando **Celery** com Django, por exemplo).
* **RNF02 - Portabilidade:** O certificado deve ser gerado no padrão **PDF/A** para garantir que a formatação nunca mude, independentemente do leitor de PDF.
* **RNF03 - Segurança:** O link de validação deve ser protegido contra "ataques de força bruta" (usar UUIDs em vez de IDs sequenciais simples na URL).
* **RNF04 - Escalabilidade:** O armazenamento dos arquivos deve ser eficiente. Em vez de salvar o PDF no banco, deve-se salvar o caminho do arquivo no servidor ou em um serviço de nuvem (S3/Azure).
* **RNF05 - Interface Responsiva:** O painel de administração deve ser acessível via mobile, permitindo que você emita um certificado rapidamente pelo celular se necessário.

---

### Arquitetura Sugerida

Para o seu perfil de desenvolvedor, a estrutura ideal seria:

1. **Backend:** FastAPI ou Django (Python).
2. **PDF Engine:** ReportLab ou WeasyPrint.
3. **Task Queue:** Celery + Redis (para enviar os e-mails sem travar o site).
4. **Storage:** Pasta local no servidor ou Amazon S3.
